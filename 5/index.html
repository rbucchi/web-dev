Riassunto

I moduli intesi come import/export sono tra le ultime feature implementate di ES6 (caniuse).

Purtroppo dipende tanto dall'env: non esistendo costrutti a tal proposito, ma solamente i 'mezzi'*, ognuno si è implementato il suo metodo e/o filosofia. È probabile che sentiate parlare di CommonJs, AMD, requirejs o ES2015 modules. Quest'ultimi stanno però stabilendo lo standard.

*Per 'mezzi' intendo le primitive js, in questo caso particolare gli scope, chiusure e l'oggetto environment (window). Le varie implementazioni citate (e spesso i ES6 modules) sono variazioni del pattern IIFE o revealing module.

TYPESCRIPTA JAVASCRIPT SUPERSET

Typescript è nato nel 2012 come superset di Javascript il cui scopo è quello di rendere JS più familiare a chi viene da linguaggi OOP e tipizzati.

Non modifica in alcun modo il runtime di JS, in nessun envirnoment, è sempre 'compilato' in JS. Le feature introdotte sono 'statiche':

  • Static Typing
  • Classi
  • Interfacce (structural typing e non nominal typing)
  • moduli
  • arrow functions
  • Enumerativi
  • Generici
  • ...
  • async/await
  • optional/default parameters

TYPESCRIPTA JAVASCRIPT SUPERSET

Set of types

Typescript è basato sullo structural typing che devia leggermente dai concetti più classici, anche se è presente in altri linguaggi come Python e GO.

Da questo concetto ne discende:

  • IsA relationship via appartenenza di metodi/attributi
    • interfaccia/tipo implicita
  • componibilità di tipi
    1. type MyType = string | null
    2. type Methods = "GET" | "POST" | "POST"

Esiste il tipo any che corrisponde a 'tutti i tipi' come evasione dal compilatore (argh). Anche l'accesso agli object literal via stringa permette l'evasione.

ANGULARA FRONTEND FRAMEWORK

Intro

Angular è un web framework, ossia un'insieme di tecnologie, librerie, strumenti e astrazioni messe a disposizione degli sviluppatori.

  1. Composizione automatica e generalistica di tecnologie (npm, typescript, rxjs)
  2. Librerie comunemente utilizzate nel web (HTTPClient, routing, modularizzazione, form, ...)
  3. Build, aggiornamenti e test preconfigurati e facilitati
  4. Gli elementi del framework astraggono i pattern del software più adatti (Component & Template come MVC, Module, Dependency injection)

Vantaggi: manutenzione facilitata (soprattutto per team estesi), tempi di sviluppo più rapidi e focus sul business model. Astrae e rende più facile lo sviluppo web (roadmap).

Svantaggi: vendor lock-in, configurazione meno flessibile, pesantezza.

ANGULARA FRONTEND FRAMEWORK

Architettura

Modulo

Contenitore di elementi (Components, Services, ...). Concettualmente simile ai namespaces, ma non è semplicemente un insieme di blocchi di codice: al suo interno gli elementi hanno un loro ciclo di vita.

Components

Implementazione logica di una porzione della pagina web. Ha un template (HTML) collegato.

Services

Singleton il cui ciclo di vita (tendenzialmente) è quello dell'applicazione. Vengono utilizzati per incapsulare logiche, mantenere dati in memoria accessibile e per comunicare tra componenti. I servizi sono iniettati nei componenti tramite la DI.

ANGULARA FRONTEND FRAMEWORK

Data Binding

Per data binding si intende la comunicazione tra la logica e il DOM (Component e Template). 


    import { Component } from '@angular/core';
    
    @Component({
        selector: 'app-counter-component',
        template: `
        <button (click)="incrementCounter()">Increment</button>
        <span>{{ counter }}</span>
        `
    })
    export class CounterComponent {
        counter = 0;
        
        incrementCounter() {
            this.counter++;
        }
    }
