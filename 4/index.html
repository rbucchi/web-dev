JAVASCRIPTTHE LANGUAGE FOR PROGRAMMING WEB PAGES

Basics, scope, functions and DOM: link

Riassunto

Dynamically and weakly typed: ovvero nessun controllo compile-time, variabili possono cambiare tipo, nessun autocomplete.

Scope: nativamente utilizza il function scope, let introduce il concetto di block scope. Quindi potete utilizzare let per rimanere su concetti 'familiari' di block scope, ma dovete sapere che una function crea il suo scope.

Functions: in JS le funzioni sono Higher-Order, ovvero sono trattate come variabili, quindi normalmente vengono assegnate o passate come parametro in un metodo.

DOM (e browser API): nell'environment 'browser' abbiamo a disposizione tutta una serie di metodi per manipolare il DOM e interagire con elementi o browser.

Equality, objects, inheritance e closures: link

Riassunto

Equality: usate === e !==

Objects: oggetti molto flessibili, largamente usati non solo per pojo, eredità o design pattern, ma anche per hashmap.

Polimorfismo: JS supporta un diverso modello di polimorfismo rispetto ai linguaggi OOP, tramite prototipo. Fortunatamente, soprattutto per chi nasce come OOP developer attualmente esiste una sintassi più chiara. JS rimane comunque OBP perché non supporta overloading.

Closures: concetto che può sembrare atipico, ma largamente utilizzato, per creare degli scope interni e/o implementare il concetto di 'privato'.

JAVASCRIPT FlavoursTHE LANGUAGE FOR PROGRAMMING WEB PAGES

  • Vanilla Javascript (normalmente < ES6)
  • Typescript, trasformazione JS in Object Oriented, con static typing e interfacce
  • Elm, puramente funzionale, static typing, immutability e module system
  • *jQuery, non è realmente un 'flavour', ma una libreria storica, che comunque ha influenzato molto la sintassi e il mondo js

A parte Vanilla JS < ES6 questi linguaggi vengono eseguiti sul browser (o su qualsiasi altri ambienti) attraverso dei 'transpiler'. (e.g.: Babel )

Javascript State of the Art

Uno dei problemi maggiori di JS come potrete aver notato è la velocità di crescita dello strumento, probabilmente dovuto a 1) mercato florido, 2) 'giovinezza', 3) paradigma di partenza (contestualmente al periodo storico) non nella norma

Fortunatamente da dopo l'adozione di ES6, quindi da qualche anno a questa parte, la proliferazione è diminuita. Sia per l'andamento del mercato, sia per il forte cambiamento che ha introdotto ES6.

Uno strumento utile e autocontenuto (anche se abbastanza autoreferenziale) è stateofjs
