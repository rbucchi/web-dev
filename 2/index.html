HTMLTHE LANGUAGE FOR BUILDING WEB PAGES

HTML è l’acronimo di HyperText Markup Language e non è un linguaggio di programmazione

È un linguaggio di markup (linguaggio con marcatori), che permette di indicare il cosa è contenuto all’interno di una pagina web

Queste indicazioni vengono date attraverso degli appositi marcatori, detti tag, che hanno la caratteristica di essere inclusi tra parentesi angolari 

<!DOCTYPE html>
<html>
<head>

<title>HTML Tutorial</title>

</head>
<body>

<h1>This is a heading</h1>

<p>This is a paragraph.</p>

</body>
</html>

Una distinzione importante tra i tag esistenti è quella tra i Block-level e gli inline level

I block-level incominciano sempre con un newline, prendono tutto lo spazio disponibile in larghezza e hanno un margine.

E.g.: div, form, ol/ul/li, section, table, h1-6

Gli inline elements invece prendono solo lo spazio 'necessario'.

E.g.: span, a, img, br, b, strong, i, button, select,

Scrivere spazi o mandare a capo all'interno del codice html non ha effetti, mentre questa formattazione viene specificata appunto dal contenuto (inteso come html) e in un secondo momento dal css.

ID e CLASS - Individuare elementi nel DOM

L'attributo id indica un elemento HTML unico all'interno della pagina. Viene identificato da #

L'attributo class invece un'insieme. Un Elemento tipicamente ha 0+ classi. Viene identificato da .

Contenuto con id="example-id"

<div class="city text">London</div>
.city {
    padding: 10px;
}

.text {
    font-family: sans-serif;
}
                
<div id="london-city">London</div>
#london-city {
    padding: 10px;
}

CSSTHE LANGUAGE FOR STYLING WEB PAGES

CSS è l’acronimo di Cascading Style Sheets ed è lo strumento per definire lo stile per i documenti web

Gli stili CSS istruiscono un browser su come il documento debba essere presentato all’utente definendone come gli elementi HTML devono essere mostrati (font, colori, disposizione, ...)

Prima del css (link) 

body {
  background-color: lightblue;
}

h1 {
  color: white;
  text-align: center;
}

p {
  font-family: verdana;
  font-size: 20px;
}

CSSTHE LANGUAGE FOR STYLING WEB PAGES

Ci sono 2 modi e mezzo per utilizzare il CSS 

1) tag style (nell'header)
<style>
    body {
        background-color: red;
    }
</style>

2) inclusione di risorsa esterna (99.99% delle volte)
<link rel="stylesheet"
        href="mystyle.css">

3) tramite attributo del tag, per debug o per persone pigre
<a href="www.google.it"
    style="color: green;">

Le precedenze

Regola numero 1: se stai guardando le precedenze, probabilmente hai complicato troppo i tuoi fogli di stile.

Precedenza per 'tipo' di foglio di stile (ordine crescente):
  1. User Agent (CSS del browser)
  2. User Normal (CSS dell'utente ???)
  3. Author Normal (CSS dell'autore del sito)
  4. Author Important
  5. User Important
Precedenza per 'specificità' (ordine crescente):
  1. ???
  2. Tag e pseudo selectors
  3. Class
  4. Id
  5. ???

!important

Menzione speciale a questo attributo che è un po' la bomba-H delle precedenze, che cerca di sovrascrivere con prepotenza massima il contenuto. Da usare con limitazione (eventuali client avrebbero dei problemi nel gestire lo stile), ma accettabile quando siamo client di una libreria (e.g.: sto usando bootstrap e proprio devo cambiare un comportamento di un tag). 

    body {
        background-color: red !important;
    }

JAVASCRIPTTHE LANGUAGE FOR PROGRAMMING WEB PAGES

Javascript è nato nel 1995 dal progetto Netscape, motivo per cui è indissolubilmente legato al Web. Ad oggi, vista la sua popolarità, è stato adottato anche a back (e.g.: nodejs) o addirittura nei db (couchdb o mongodb)

Questo vuol dire in realtà molto, perché a parte i paradigmi, vuol dire che:

  • Javascript, dom e window sono strettamente legati
  • Adozione obbligata di modello asincrono (prima callback, poi Promise, poi async)
  • Supporto per funzionalità specifiche per animazioni, ma non per il file system o rete
  • ...

JAVASCRIPTTHE LANGUAGE FOR PROGRAMMING WEB PAGES

È un linguaggio di programmazione interpretato, first class function, proto-based, multi-paradigm, single-threaded, dynamic. 

<p id="my">Click the button</p>

<button onclick="myFun()">Click Me!</button>

<script>

    function myFun() {
        var x = document.getElementById("my");
        x.style.fontSize = "25px";
        x.style.color = "red";
    }

</script>

Javascript vs Java

Problema n.0 di Javascript, viene associato ad un altro linguaggio con cui centra poco. Javascript, in principio Mocha poi Livescript e JS, è stato creato molto velocemente e la scusa è "cercavamo un nome e la sintassi ricorda Java". Probabilmente è stata anche una mossa di marketing visto che negli anni 90 Java era sulla cresta dell'onda.

Ad oggi però questo crea il problema non trascurabile dei confronti:

  1. Object Oriented vs Object Based
  2. Classi vs Object (solo poi supporto classi)
  3. Non funzionale (solo poi supporto) vs funzionale
  4. Sintassi simili, cambiate nel tempo (== e ===)

Javascript vs ECMAScript

ECMAScript è lo standard, Javascript l'implementazione. Semplificando possiamo trattarli come sinonimi, ma dobbiamo sapere che nel web ci sono diverse versioni supportate.

  1. ECMAScript 1-3 ES1-3 anni 90
  2. ECMAScript 5 ES5 2009
  3. ECMAScript 6 ES6 2015
  4. ECMAScript 7 ES7 2016
  5. ECMAScript 8 ES8 2017
  6. ECMAScript 9 ES9 2018
